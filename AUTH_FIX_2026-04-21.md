# AI HR Pilot — Auth Fix (2026-04-21)

## Summary

**Problem:** Google OAuth broken. Email sign-up + free trial broken on https://aihrpilot.com.

**Root causes (all confirmed via Clerk API inspection, CSP header inspection, DNS, and code audit):**

1. **CSP blocked Cloudflare Turnstile.** Clerk's production instance uses Turnstile for bot protection (`captcha_provider: turnstile`, `widget_type: smart`). The widget loads its script + iframe from `https://challenges.cloudflare.com`. The app's Content-Security-Policy did not whitelist this host, so Turnstile silently failed to mount → the email sign-up form had no working CAPTCHA → submit hung with no user feedback.
2. **CSP blocked the Clerk Account Portal (`accounts.aihrpilot.com`) and Clerk worker DNS (`*.clerk.services`).** Google OAuth redirects through the Account Portal; the pop-up window was being blocked.
3. **`NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL` and `NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL` were MISSING from Vercel Production.** Clerk fell back to its Account Portal default (`https://aihrpilot.com`, the marketing page). So the "free trial" was technically creating accounts but bouncing users back to the landing page instead of dropping them in the product.

## What YOU MUST do in the Clerk Dashboard (required to finish the fix)

Go to https://dashboard.clerk.com → select the AI HR Pilot production app (`pk_live_Y2xlcmsuYWlocnBpbG90LmNvbSQ`).

### 1. Bot Protection: flip from "Smart" to "Invisible" (HIGH priority)

**Configure → Attack Protection → Bot Protection** → set to **Invisible**. Save.

Why: "Smart" shows a visible Turnstile checkbox on sign-up. "Invisible" still blocks bots but does not add visual friction. Real buyers will abandon at the first CAPTCHA on a paid-trial sign-up.

### 2. Verify Google OAuth has REAL custom credentials (CRITICAL for OAuth)

**User & Authentication → Social Connections → Google** → click it → inspect settings.

On a production Clerk instance (any `pk_live_*` key), Google OAuth CANNOT use Clerk's shared dev OAuth credentials. You must have a real Google Cloud Console OAuth 2.0 Client with:
- Authorized redirect URI: `https://clerk.aihrpilot.com/v1/oauth_callback`
- Authorized JavaScript origin: `https://aihrpilot.com`

If the connection says "Use custom credentials" toggle is OFF, or the Client ID / Secret fields are empty, Google OAuth will hard-fail on production. To fix:

1. Go to https://console.cloud.google.com/apis/credentials → pick or create a project → **+ CREATE CREDENTIALS** → **OAuth client ID** → type **Web application**.
2. **Authorized JavaScript origins:** `https://aihrpilot.com`, `https://clerk.aihrpilot.com`, `https://accounts.aihrpilot.com`
3. **Authorized redirect URIs:** `https://clerk.aihrpilot.com/v1/oauth_callback`
4. Copy the Client ID and Client Secret.
5. OAuth consent screen must be **Published** (not Testing), or Google will only let your listed test users through.
6. Back in Clerk → paste Client ID + Secret → enable "Use custom credentials" → Save.

### 3. Verify Account Portal redirects (medium, since env vars now override)

**Paths** → set:
- After sign-in URL: `/dashboard`
- After sign-up URL: `/onboarding`

The Vercel env vars I added override these at the client-SDK level, but keeping them consistent prevents the Account Portal fallback from sending users to the marketing homepage.

## How to verify the full fix is working

1. Open https://aihrpilot.com/sign-up in an **incognito** window (avoids cached stale builds).
2. Email sign-up: enter `test+<timestamp>@portlev.com`, password, name → should go through with no visible CAPTCHA, land on https://aihrpilot.com/onboarding.
3. Google sign-up: click "Continue with Google" button → Google consent screen → redirected back → lands on /onboarding.
4. Sign out, sign back in the same way → lands on /dashboard.

If any of these fail:
- Open DevTools → Console. CSP violations will name the exact blocked host → add to `next.config.mjs` CSP.
- Open DevTools → Network → filter "oauth". A 400 from Google = Google OAuth credentials not set in Clerk. A 307 back to aihrpilot.com without landing on /dashboard = Clerk Account Portal config still overriding the env var.
